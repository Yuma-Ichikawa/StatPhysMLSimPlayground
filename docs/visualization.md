# Visualization

All plotting lives in `statphys.vis`. Static plotters share a consistent publication style; animations export to GIF (Pillow) or MP4 (ffmpeg).

## Plotters

| Class / function | Purpose |
|---|---|
| `ComparisonPlotter` | Theory vs experiment: order parameters and generalization error |
| `OrderParamPlotter` | Order-parameter curves |
| `PhaseDiagramPlotter`, `compute_phase_grid` | 2D phase diagrams over parameter grids |
| `DynamicsPlotter` | ODE flow fields, nullclines, speed maps, phase portraits in the $(m, q)$ plane |
| `OverlapMatrixPlotter` | $M$/$Q$/$R$ overlap-matrix heatmaps and specialization snapshots over time |
| `SweepPlotter` | Parameter sweeps, theory-vs-experiment scatter diagnostics, replica convergence checks |
| `plot_from_replica_results`, `plot_from_online_results` | One-call default figures from simulation results |
| `apply_paper_style` | Matplotlib rcParams for publication-quality output |

### Phase portraits

```python
from statphys.vis import DynamicsPlotter
from statphys.theory.online import GaussianLinearMseEquations

eqs = GaussianLinearMseEquations(rho=1.0, lr=0.5)
plotter = DynamicsPlotter()

# Vector field + nullclines
plotter.plot_flow_field(eqs, m_range=(0, 1.2), q_range=(0.01, 1.5))

# Flow field + theory trajectories + per-seed experiment trajectories
plotter.plot_phase_portrait(eqs, result=results)
```

### Overlap matrices (specialization)

```python
from statphys.vis import OverlapMatrixPlotter

plotter = OverlapMatrixPlotter()
plotter.plot_matrix(M, title="student-teacher overlaps", annotate=True)
plotter.plot_specialization_over_time(M_snapshots, t_values)
```

## Animations

`statphys.vis.animation` produces matplotlib `FuncAnimation` objects:

| Function | Shows |
|---|---|
| `animate_learning_curve(t, trajectories, theory=..., logx=, logy=)` | Order parameters (and/or errors) being drawn over time, with static theory curves; log axes for epoch-scale (grokking-style) plots |
| `animate_phase_plane(m_traj, q_traj, equations=...)` | A point moving through the $(m, q)$ plane on top of the ODE flow field |
| `animate_overlap_matrix(matrices, t_values=...)` | The $M$ overlap matrix evolving during training (specialization emerging) |
| `animate_decision_boundary(X, y, weights, metric_values=...)` | A 2D linear classifier's decision line rotating into place on labelled data (e.g. Gaussian-mixture classification) — the most intuitive, textbook-style animation |
| `save_animation(anim, path, fps=20)` | Save; the writer is chosen from the extension (`.gif` → pillow, `.mp4` → ffmpeg) |

```python
from statphys.vis import animate_phase_plane, save_animation
from statphys.theory.online import GaussianLinearMseEquations

eqs = GaussianLinearMseEquations(rho=1.0, lr=0.5)
anim = animate_phase_plane(m_traj, q_traj, equations=eqs)
save_animation(anim, "dynamics.gif", fps=20)
```

A grokking-style animation with log axes:

```python
from statphys.vis import animate_learning_curve, save_animation

anim = animate_learning_curve(
    epochs, {"train error": train_err, "test error": test_err},
    xlabel="epoch", ylabel="error", logx=True, logy=True,
)
save_animation(anim, "grokking.gif", fps=20)
```

A decision-boundary animation for 2D classification data (e.g.
`GaussianMixtureDataset`, see [order_parameters.md](order_parameters.md#5-gaussian-mixture-classification-an-exactly-checkable-epsilon_g)):

```python
from statphys.vis import animate_decision_boundary, save_animation

anim = animate_decision_boundary(
    X, y, weight_snapshots, metric_values=test_errors, metric_name=r"$\epsilon_g$",
)
save_animation(anim, "boundary.gif", fps=20)
```

## Experiment results

`ExperimentResult.plot()` (from `statphys.experiment`) draws seed-averaged metrics with error bands and log-axis options:

```python
result = exp.run_sample_complexity(alphas=[1, 2, 4, 8, 16])
fig, ax = result.plot(logy=True)
```
